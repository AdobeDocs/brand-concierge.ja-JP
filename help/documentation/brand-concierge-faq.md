# よくある質問

Brand Conciergeに関するよくある質問への回答については、この節を参照してください。

## 全般

### Brand Conciergeとチャットボットの違い

Brand Conciergeは、従来のチャットボットとは一線を画しています。スクリプト形式の回答や一般的な web 結果ではなく、組織のコンテンツや顧客データに基づいて特別にトレーニングを受けた生成 AI を活用するからです。 これにより、アシスタントは個々の顧客行動に基づいてパーソナライズされた回答を提供し、Adobeのツールやデータと深く統合し、各インタラクションから継続的に学習し、基本的なキーワードマッチングを超えて顧客意図を正確に解釈できます。

### B2C と B2B の両方にBrand Conciergeを使用できますか？

はい。次に、使用例を示します。

* **B2C:** 製品検出、買い物サポート、カスタマーサポート、パーソナライズされたレコメンデーション。
* **B2B:** ガイド付き評価、機能比較、ミーティングのスケジュール、営業担当者のルーティング、コンサルテーションの予約。

### Brand Conciergeを使用できる業界は何ですか？

Brand Conciergeは、小売および e コマース、旅行およびホスピタリティ、金融サービス、ヘルスケア（コンプライアンス制御）、メディアおよびエンターテイメント、テクノロジーおよびソフトウェアなど、様々な業界で使用できます。 基本的に、お客様が情報を見つけて意思決定を行うのを支援する業界であれば、Brand Conciergeを実装することでメリットを得ることができます。

## データとプライバシー

### 顧客データは安全ですか？

はい。Brand Conciergeでは、GDPR および CCPA への準拠、Adobeの安全なインフラストラクチャでのデータ処理、データ使用の制御を提供、暗号化と監査ログを通じた会話の保護により、お客様のデータを安全に保護します。

すべての会話は、サードパーティのサーバーではなく、プロパティで行われます。

### 接続できるデータソースは何ですか？

次のタイプのデータソースをBrand Conciergeに接続できます。

| Data Sourceのタイプ | 使用可能なソース /詳細 |
|------------------|---------------------------|
| **製品とコンテンツ** | 商品カタログ <br> 在庫システム <br> ナレッジベースおよびドキュメント <br>CSV URL を使用した web サイトコンテンツのアップロード <br>Adobe Experience Managerコンテンツ <br>Adobe Commerceデータ |
| **顧客データ** | Adobe Experience Platform プロファイル <br>Adobe Analyticsの動作データ <br> ファーストパーティ顧客属性 <br> サードパーティ API （設定済み） |
| **CSV ファイル形式** | Web サイトの URL を含む 1 つの列 <br>Brand Conciergeが URL をクロールしてコンテンツを自動抽出 <br> ステータスの更新をリアルタイムで処理 <br> コンテンツ領域ごとに複数の CSV ファイルをアップロードできます |

すべてのデータは、ガバナンスルールに従います。

### 顧客はパーソナライゼーションをオプトアウトできますか？

はい。オプトアウトした顧客は、行動パーソナライゼーションを行わずに役立つ回答を受け取ります。 プライバシーポリシーに合わせてオプトアウト処理を設定します。

## 設定と制御

### ブランドの声を制御するにはどうすればよいですか？

トーン（フォーマルからカジュアルまで）、言語（シンプルからテクニカルまで）、パーソナリティ（例：役に立つ、熱狂的、プロフェッショナルなど）などの要素を設定することで、UI で直接ブランドの声を制御できます。 さらに、テンプレートや例を使用して応答パターンを定義し、コンプライアンスルールや境界を適用するガードレールを確立することもできます。 Adobeの参照プロンプトから始めて、ブランドの一意の ID を反映するようにこれらの設定を調整します。

### Brand Conciergeが質問に答えられない場合はどうなりますか？

フォールバックビヘイビアーを設定すると、Brand Conciergeが質問に答えることができない場合の対処方法を指定できます。 オプションには、優雅な「私はそれを助けることができない」メッセージを表示する、代替の質問を提案する、セルフサービスリソースにリンクする、または自動的に人間のエージェントに問い合わせをエスカレーションするなどがあります。 ブランドに最適なものを選択します。

### ビジュアルデザインをカスタマイズできますか？

はい。次のようなすべての視覚要素をカスタマイズします。

* カラーとブランディング
* フォントとタイポグラフィ
* ボタンのスタイル
* ウィジェットの配置
* カードレイアウト
* 応答の書式設定

SDK には、デフォルトコンポーネントと完全なカスタマイズオプションが用意されています。

### セットアップにはどのくらいの時間がかかりますか？

設定の長さは、実装のタイプによって異なる場合があります。 既存の製品カタログ、標準 FAQ コンテンツ、デフォルト設定を含む基本実装は、設定に約 3～5 日かかる場合があります。 一方、カスタム統合、広範なパーソナライゼーション、複雑なワークフロー、カスタムコンプライアンスルールによる高度な実装の完了には、約 2～4 週間かかる場合があります。

### プレビューとテストの仕組み

Brand Conciergeには、組み込みのテストツールが用意されています。

| 試験治具 | 機能 |
|--------------|----------|
| **プレビューモード** | 顧客の会話をシミュレート <br> 設定をリアルタイムで調整 <br> 変更を即時に確認 <br> プレビューリンクをチームと共有 |
| **テスター表示** | 親指の上下による回答の評価 <br>4 つのカテゴリにわたる構造化されたフィードバックの提供 <br> 詳細なコメントの追加 <br> ダッシュボードでのフィードバックの追跡 |

顧客にデプロイする前に、すべてのテストが行われます。

### 顧客はチームとのミーティングをスケジュールできますか？

はい、顧客は Meeting Booking スキルを使用してチームとのミーティングをスケジュールできます。 この機能を有効にするには、スキル構成でスキルをアクティブにし、アクティブ化の目的（「営業と話す」など）を定義し、カレンダーまたはスケジュール システムを接続して、空き時間と会議の種類を設定します。 設定が完了すると、お客様は会話中にミーティングをリクエストできます。Brand Conciergeを使用すると、お客様はチャットを離れることなく、スケジュールプロセスを容易に実行できます。

### エンジニアリングを迅速に行うのは誰ですか？

Adobe コンサルタントは、次のようなバックグラウンドで迅速なエンジニアリングを行います。

1. 設定に関する質問には、スキル ページで回答します。
1. 製品に関する知識やビジネスルールを提供し、キーワードを回避します。
1. 入力を送信します。
1. Adobe コンサルタントは、お客様の回答を使用して最適化されたプロンプトを設計します。
1. 変更はコンシェルジュに自動的に反映されます。

これにより、コンシェルジュは、特定のブランド要件を維持しながら、ベストプラクティスの AI プロンプトパターンを使用できます。

## パフォーマンスと分析

### 成功を測定するにはどうすればよいですか？

成功を測定するには、Brand Concierge ダッシュボードを使用します。 ダッシュボードを使用して、次のような指標を追跡します。

| 指標 | トラッキング対象 |
|--------|----------------|
| **エンゲージメント** | 会話のボリューム、セッションの長さ |
| **満足度** | センチメントスコア、フィードバック評価 |
| **コンバージョン** | 補助と補助なしの購入率 |
| **トピック** | よくある質問とリクエスト |
| **ハンドオフ** | エスカレーション率と理由 |
| **パフォーマンス** | 応答精度、分解能 |

Adobe Analyticsと統合して、より詳細な分析を行うこともできます。

### センチメントが低下した場合はどうすればよいですか？

センチメントが低下した場合は、最近の失敗したクエリを確認し、コンテンツのギャップをチェックし、ネガティブフィードバックを分析し、適切なトーンをテストし、技術的な問題があれば検証することで、根本原因を調査します。 根本的な原因が特定されたら、速やかに対処し、改善に向けた監視を継続します。

## 統合と技術

### 他のAdobe製品は必要ですか？

いいえ。ただし、パフォーマンスが向上します。

| 統合オプション | 機能 |
|-------------------|--------------|
| **スタンドアロン** | 製品カタログとコンテンツの操作 |
| **Adobe Experience Platformの場合** | 統合された顧客プロファイル <br> 高度なパーソナライゼーション <br> クロスチャネルの一貫性 |
| **Adobe Commerceの場合** | リアルタイムの在庫 <br> 注文履歴 <br> 買い物かごの統合 |
| **Adobe Experience Managerの場合** | コンテンツ管理 <br> 動的更新 <br> マルチサイトのサポート |

### 自分のサイトがAdobeにない場合はどうなりますか？

Brand Conciergeは、あらゆるプラットフォームで機能します。 JavaScript SDKは任意の web サイトと統合され、モバイル SDK は任意のアプリバックエンドで動作します。

### エージェントのハンドオフはどのように機能しますか？

エージェントのハンドオフがトリガーされると、Brand Conciergeは、すべての会話の履歴、お客様のプロファイルと ID、識別された意図、取り上げられた製品の詳細、エージェントへの解決の試みについて転送します。 これにより、エージェントは完全なコンテキストを持ち、顧客が情報を繰り返す必要なく、シームレスに会話を継続できます。

### 複数の言語をサポートできますか？

はい。顧客ベースに基づいて、アシスタントごとに言語サポートを設定します。 Brand Conciergeはお客様の言語を検出し、それに応じて応答します。
